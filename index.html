🧠
① Report Planning
The planner analyzes the topic, generates 4 search queries, fetches web context, and outputs a structured section plan with names, descriptions and research flags.
⚡
② Parallel Research
Each section spawns its own worker via the Send() API — generate queries → search web → write section, all fully concurrent.
✍️
③ Final Sections
Introduction and Conclusion are written in parallel after all research sections complete — synthesizing the collected content rather than doing independent web searches.
📄
④ Report Compilation
All sections are stitched into a Markdown wiki document with proper formatting. Output is production-ready and downloadable as .md.
